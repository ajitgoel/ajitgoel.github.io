# Guidelines for Content Creation

When creating or modifying blog posts on this site, follow these strict rules:

### 1. No Hero/Heading Images
- **NEVER** add a hero image or a heading image at the top of a blog post.
- Technical diagrams (like Mermaid) or screenshots within the content are encouraged, but decorative "vibe" images at the start are prohibited.

### 2. Write Like a Human (Not an AI)
To ensure the blog feels personal and authentic:
- **Avoid Flowery AI Phrasing**: Avoid words like "delve," "unlocking," "harnessing," "in the modern landscape," "testament to," or "embarking on a journey."
- **Use First Person**: Speak from personal experience ("I found that...", "We struggled with...", "I decided to...").
- **Be Direct**: Get to the point quickly. Start with the problem or the context instead of a high-level philosophical introduction.
- **Short Paragraphs**: Keep things scannable and punchy.
- **Technical Tone**: Maintain a professional but conversational tone. Think "peer-to-peer technical advice" rather than "marketing copy."
- **Personal Anecdotes**: Include specific details about why a project was started or what the actual "on-the-ground" challenge was.

### 3. Formatting
- Use semantic HTML (h2, h3 for headings).
- Use `pre` and `code` tags for code snippets.
- Use `mermaid-container reveal` for diagrams to maintain the site's animation system.
